## 実行結果

全 8 フェーズが正常に完了。

### Phase 1: iso-download
- Debian 13.3 netinst ISO は既にダウンロード済み (sha256 一致)

### Phase 2: preseed-generate
- `generate-preseed.sh` で preseed-generated-s4.cfg を生成
- ネットワーク: auto、ディスク: /dev/nvme0n1、コンソール: ttyS1

### Phase 3: iso-remaster
- ISO リマスター実行 (前回の preseed hash なし)
- 出力: /var/samba/public/debian-preseed-s4.iso (762MB)
- preseed hash 保存済み

### Phase 4: bmc-mount-boot
- サーバを ForceOff してから開始
- VirtualMedia マウント + Redfish verify 成功
- POST code API が stale 値 (0x01) を返す既知の問題が発生
- KVM スクリーンショットで POST 完了を確認
- ATEN Virtual CDROM の Boot ID: Boot0011
- BootNext 設定 + パワーサイクルで ISO ブート開始

### Phase 5: install-monitor
- SOL 監視で Debian インストールを追跡
- ステージ進行: LOADING_COMPONENTS (2.8min) -> CONFIGURING_APT (3.0min) -> INSTALLING_SOFTWARE (6.0min) -> INSTALLING_GRUB (7.2min) -> POWER_DOWN (8.0min)
- インストール完了: 約 8 分

### Phase 6: post-install-config
- Power On 後、POST 92 (PCI Bus Enumeration) でスタック発生
- ForceOff -> 20s -> On のリカバリで復旧
- SOL 経由で root SSH 鍵、PermitRootLogin、sudoers、静的 IP を設定
- SSH 接続確認成功

### Phase 7: pve-install
- pre-reboot: PVE リポジトリ追加、パッケージ更新、PVE カーネルインストール
- リブート後、80 秒で SSH 復旧 (POST 92 は未発生)
- インターネット接続確認 OK (deb.debian.org 到達可能)
- post-reboot: proxmox-ve フルインストール、Debian カーネル削除
- 最終リブート時に POST 92 再発 -> ForceOff リカバリで復旧

### Phase 8: cleanup
- VirtualMedia アンマウント、Boot Override リセット
- 最終検証: OS、PVE、カーネル、ネットワーク、Web UI すべて正常

## 特記事項

1. **POST 92 スタック**: 4号機固有の既知問題が 2 回発生 (Phase 6 初回ブート、Phase 7 最終リブート)。いずれも ForceOff -> 20s -> On のリカバリで復旧。
2. **POST code API stale 値**: POST code が 0x01 のまま更新されない既知問題。KVM スクリーンショットで実際の状態を確認して対処。
3. **合計所要時間**: 約 60 分。bmc-mount-boot (POST stale 対応) と pve-install (パッケージDL + POST 92 リカバリ) がボトルネック。
