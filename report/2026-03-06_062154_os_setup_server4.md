## 特記事項

### Phase 3: remaster-debian-iso.sh の出力ファイル名バグ

`remaster-debian-iso.sh` が Docker コンテナ内で出力ファイル名を `debian-preseed.iso` にハードコードしていたため、`debian-preseed-s4.iso` として指定しても正しいファイル名で出力されなかった。

**修正内容**: `OUTPUT_BASENAME` 環境変数をコンテナに渡し、コンテナ内でも指定されたファイル名を使用するように変更。

### Phase 5: preseed インストーラの異常動作

Debian インストーラが GRUB "Automated Install" メニューから起動し、SOL 経由でインストーラ TUI の進行が約 2 時間確認された。しかし、インストール完了後にシステムをパワーサイクルすると、以前のインストール (3月1日) の Debian + PVE 9.1.6 が起動した。

**推定原因**:
- preseed ファイルが ISO から正しく読み込まれなかった可能性
- インストーラが VGA コンソール上で対話的に実行され、preseed が適用されなかった可能性
- SOL に表示されたインストーラ TUI の status bar は進行していたが、実際にはディスクへのインストールが行われなかった

**結果**: 既存の正常な PVE インストールが維持されていたため、Phase 6-7 をスキップして Phase 8 の最終検証に進んだ。

### playwright のインストール

Phase 5 の長時間待機中に KVM スクリーンショットを取得するため、`playwright` を `.venv` にインストールした。以後のセッションでも使用可能。
