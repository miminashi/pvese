## 再現方法

### Step 1: Debian 13 netinst ISO ダウンロード

```bash
curl -L -o /var/samba/public/debian-13.3.0-amd64-netinst.iso \
  https://cdimage.debian.org/debian-cd/current/amd64/iso-cd/debian-13.3.0-amd64-netinst.iso
```

SHA256: `c9f09d24b7e834e6834f2ffa565b33d6f1f540d04bd25c79ad9953bc79a8ac02`

### Step 2: preseed.cfg 更新

ファイル: `preseed/preseed.cfg`

主要変更 (Debian 12 → 13):
- コメント行のバージョン番号を更新
- **console 順序反転**: `console=tty0 console=ttyS1,115200n8` (ttyS1 をプライマリに)

### Step 3: ISO リマスタースクリプト更新

ファイル: `scripts/remaster-debian-iso.sh`

主要変更:
- Docker イメージ: `debian:bookworm` → `debian:trixie`
- デフォルト ISO パス: Debian 13.3.0
- GRUB/isolinux カーネルパラメータ: console 順序反転 + `quiet` 除去

### Step 4: ISO リマスター実行

```bash
scripts/remaster-debian-iso.sh
```

出力: `/var/samba/public/debian-preseed.iso` (755MB)

### Step 5: VirtualMedia マウント & ブート

BMC CGI API を使用:
```
# BMC ログイン
POST /cgi/login.cgi  name=claude&pwd=Claude123

# CSRF トークン取得
GET /cgi/url_redirect.cgi?url_name=topmenu → CSRF_TOKEN ヘッダ値を取得

# ISO パス設定
POST /cgi/op.cgi  (Header: CSRF_TOKEN: <token>)
  op=config_iso&host=10.1.6.1&path=\public\debian-preseed.iso&user=&pwd=

# マウント
POST /cgi/op.cgi  op=mount_iso

# Boot Override (Redfish)
PATCH /redfish/v1/Systems/1
  {"Boot":{"BootSourceOverrideEnabled":"Once","BootSourceOverrideTarget":"Cd","BootSourceOverrideMode":"UEFI"}}

# 電源サイクル
POST /redfish/v1/Systems/1/Actions/ComputerSystem.Reset  {"ResetType":"ForceOff"}
(8秒待機)
POST /redfish/v1/Systems/1/Actions/ComputerSystem.Reset  {"ResetType":"On"}
```

**注意**: CSRF トークンのヘッダ名は `CSRF_TOKEN` (前回 `X-CSRFTOKEN` で失敗した)

### Step 6: インストール監視 (SOL)

pexpect でバックグラウンド SOL セッションを起動:

child = pexpect.spawn("ipmitool -I lanplus -H 10.10.10.24 -U claude -P Claude123 sol activate")

### Step 7: 初期設定

インストーラの preseed late_command が失敗したため、SOL 経由で root ログインして設定:

1. `/etc/sudoers.d/debian` — debian ユーザーに NOPASSWD sudo
2. `/root/.ssh/authorized_keys` — SSH 公開鍵
3. `/etc/ssh/sshd_config` — `PermitRootLogin yes`
4. `/etc/network/interfaces` — eno2np1 に 10.10.10.204/8 static

### Step 8: Proxmox VE 9 インストール

```bash
# /etc/hosts 設定
sed -i "s/127.0.1.1/10.10.10.204/" /etc/hosts

# PVE 9 リポジトリ追加
echo "deb [arch=amd64] http://download.proxmox.com/debian/pve trixie pve-no-subscription" \
  > /etc/apt/sources.list.d/pve-install-repo.list
wget -q https://enterprise.proxmox.com/debian/proxmox-release-trixie.gpg \
  -O /etc/apt/trusted.gpg.d/proxmox-release-trixie.gpg

# cdrom リポジトリ無効化 + apt update + full-upgrade
sed -i '/^deb cdrom:/s/^/#/' /etc/apt/sources.list
apt-get update && apt-get full-upgrade -y

# PVE カーネル
apt-get install -y proxmox-default-kernel

# GRUB シリアル設定更新
# /etc/default/grub:
#   GRUB_CMDLINE_LINUX_DEFAULT=""
#   GRUB_CMDLINE_LINUX="console=tty0 console=ttyS1,115200n8"
#   GRUB_TERMINAL="console serial"
#   GRUB_SERIAL_COMMAND="serial --unit=1 --speed=115200 --word=8 --parity=no --stop=1"
update-grub && reboot

# PVE カーネルで再起動後
apt-get install -y proxmox-ve postfix open-iscsi chrony

# Debian カーネル削除
apt-get remove -y linux-image-amd64 'linux-image-6.12*'
update-grub

# locale 修正
sed -i "s/# en_US.UTF-8/en_US.UTF-8/" /etc/locale.gen && locale-gen
```

## 問題と対策

### 1. BMC CSRF トークンのヘッダ名

- **症状**: `X-CSRFTOKEN` ヘッダで "Token Value is not matched" エラー
- **原因**: 正しいヘッダ名は `CSRF_TOKEN`（前回の成功セッションのログから発見）
- **対策**: `-H "CSRF_TOKEN: <token>"` 形式で送信

### 2. preseed late_command 失敗

- **症状**: sudoers 設定、SSH PermitRootLogin 設定が反映されていない
- **原因**: Debian 13 でも late_command の `in-target` が失敗
- **対策**: SOL 経由で root ログインし手動設定。preseed の late_command の信頼性向上は今後の課題

### 3. preseed poweroff が効かない

- **症状**: インストール完了後に電源オフにならず、メインメニューに戻った
- **原因**: `debian-installer/exit/poweroff` が Debian 13 インストーラで期待通り動作しなかった可能性
- **対策**: VirtualMedia アンマウント後に手動で ForceOff → On

### 4. POST コード 92 スタック

- **症状**: 再起動後に PCI Bus Initialization (コード 92) で長時間停止
- **対策**: ForceOff → 15秒待機 → On の完全電源サイクルで解消

### 5. NIC 名の一貫性

- **改善点**: 前回は Debian カーネル (6.1) → PVE カーネル (6.8) で NIC 名が `eno1` → `eno1np0` に変化したが、今回は Debian 13 カーネル (6.12) の時点で既に `eno1np0` だったため、PVE カーネル (6.17) への移行時に名前変更が発生しなかった
