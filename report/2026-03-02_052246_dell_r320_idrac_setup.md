# DELL PowerEdge R320 (7号機) iDRAC SSH セットアップレポート

- **実施日時**: 2026年3月2日 05:22

## 前提・目的

pvese クラスタに7号機 (DELL PowerEdge R320) を追加するための初期セットアップ。
iDRAC SSH 鍵認証を構成し、リモート管理の基盤を整備する。

- 背景: 既存クラスタは Supermicro X11DPU 3台 (4-6号機)。異なるベンダーのサーバを追加
- 目的: iDRAC SSH 接続を確立し、config/server7.yml を作成する
- 前提条件: iDRAC が 10.10.10.120 でネットワーク到達可能であること

## 再現方法

### 1. SSH 鍵生成

Ed25519 を最初に試みたが iDRAC が未対応だったため RSA 2048 で生成:

```sh
ssh-keygen -t rsa -b 2048 -C "pvese-idrac" -f ~/.ssh/idrac_rsa -N ""
```

### 2. SSH config 追記

`~/.ssh/config` に以下を追加:

```
Host idrac7
  HostName 10.10.10.120
  User claude
  IdentityFile ~/.ssh/idrac_rsa
  IdentitiesOnly yes
  KexAlgorithms +diffie-hellman-group14-sha1,diffie-hellman-group1-sha1
  HostKeyAlgorithms +ssh-rsa,ssh-dss
  PubkeyAcceptedAlgorithms +ssh-rsa
```

### 3. iDRAC Web UI で公開鍵登録

`~/.ssh/idrac_rsa.pub` の内容を iDRAC Web UI (`https://10.10.10.120`) の claude ユーザに登録。

### 4. 接続テスト

```sh
ssh idrac7 "racadm getsysinfo"
```

接続成功。`racadm getsysinfo` でサーバ情報を取得できることを確認。

## 備考

- iDRAC は Ed25519 未対応。RSA 2048 鍵を使用
- DELL PowerEdge R320 は Supermicro X11DPU とは BMC が異なるため、既存の `scripts/bmc-*.sh` スクリプトはそのままでは使えない
- NIC 名、ディスクパス等は OS インストール後に `config/server7.yml` を更新する必要がある
- iDRAC SSH config に古い鍵交換アルゴリズム (`diffie-hellman-group14-sha1` 等) を許可設定済み
