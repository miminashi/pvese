## Issue #38: IPoIB リブート後に自動起動しない

### 原因

`ib-setup-remote.sh --persist` が書く `/etc/network/interfaces.d/ib0` の `pre-up modprobe ib_ipoib` だけでは、networking.service がインターフェースを起動する時点でモジュールがロードされていない場合がある。

### 修正内容 (`scripts/ib-setup-remote.sh`)

1. **`/etc/modules-load.d/ib_ipoib.conf` の作成**: `systemd-modules-load.service` が networking.service より前に `ib_ipoib` モジュールをロードする
2. **`pre-up sleep 2` の追加**: udev リネーム (ib0 → ibpXXsX) の完了を待つ。既存の runtime 側 `sleep 2` (line 57) と同じ待機時間

## Issue #39: LINBIT リポジトリ + enterprise.sources 除去

### 原因

`pve-setup-remote.sh` の post-reboot フェーズが enterprise リポジトリ (`.list` と `.sources` の両方) を削除せず、LINBIT リポジトリのセットアップも行わないため、os-setup 後に手動対応が必要だった。

### 修正内容 (`scripts/pve-setup-remote.sh`)

1. **enterprise リポジトリ除去** (常時実行): `pve-enterprise.list` + `pve-enterprise.sources` を `rm -f` で削除
2. **`--linstor` フラグ追加**: LINBIT GPG 鍵ダウンロード、リポジトリ追加、DRBD/LINSTOR パッケージインストールを実行 (オプション)
3. **usage にフラグ説明追加**

### SKILL.md 更新

- Phase 7 ステップ 4: `--linstor` フラグの使用例と説明を追加
- Phase 8 ステップ 6: `--persist` が `modules-load.d` も書く旨を注記
