# SX6036 IB スイッチ ファン制御調査レポート

- **実施日時**: 2026年4月13日 21:29 - 22:02 (JST)

## 前提・目的

Mellanox SX6036 InfiniBand スイッチのファン速度制御方法を調査し、ファン速度を手動で変更できるか確認する。

- **背景**: SX6036 のファンは常時 ~9500-10000 RPM (メインファン) / ~11000 RPM (電源ファン) で動作しており、騒音低減やファン制御の自動化の可能性を調べたい
- **目的**: MLNX-OS CLI、configure terminal、SNMP のいずれかでファン速度を制御できるか網羅的に調査する
- **前提条件**: 4号機 (10.10.10.204) からの USB シリアル接続でスイッチコンソールにアクセス可能

## 調査結果

### 結論: **ファン速度の手動制御は不可能**

MLNX-OS 3.6.8012 (SwitchX-2 プラットフォーム) にはファン速度を制御する CLI コマンドが存在しない。ファン速度はファームウェアが温度センサーに基づいて自動制御しており、ユーザが変更する手段はない。

### 2. CLI コマンド調査

#### operator mode (>) — 読み取りのみ

| コマンド | 結果 |
|---------|------|
| `show fan` | ファン RPM + ステータス表示。制御パラメータなし |
| `show system` | `% Incomplete command` (サブコマンドが必要) |
| `show module` | FAN/PS1/PS2 の status=ready を表示 |
| `show voltage` | 全電圧正常。ファン制御関連なし |

#### enable mode (#) — 読み取り + 管理

| コマンド | 結果 |
|---------|------|
| `show running-config` | ファン関連設定なし (NIC IP, SM 設定のみ) |
| `show health` | ファン健全性アラート表示。制御パラメータなし |
| `show chassis` | `% Unrecognized command` |
| `show environment` | `% Unrecognized command` |

#### configure terminal mode ((config) #) — 設定変更

| コマンド | 結果 |
|---------|------|
| `fsc mode manual` | `% Unrecognized command "fsc"` |
| `fan-speed set 50` | `% Unrecognized command "fan-speed"` |
| `fan speed auto` | `% Unrecognized command "fan"` |
| `system fan-speed 50` | `% Unrecognized command "fan-speed"` |
| `health fan speed min 5000` | `% Unrecognized command "fan"` |

### 3. SNMP 調査

SNMP v2c (community: `public`) 経由で以下を確認:

- **ENTITY-MIB**: FAN/FAN/F1-F4, PS1/FAN/F1, PS2/FAN/F1 がエンティティとして登録
- **ENTITY-SENSOR-MIB**: ファン RPM 値を読み取り可能 (entPhySensorValue)
  - F1=10020, F2=9510, F3=10020, F4=9510, PS1=11490, PS2=10830
- **Mellanox Enterprise OID (1.3.6.1.4.1.33049)**: IB ポートインデックスのみ。ファン制御 OID なし
- **書き込み可能な OID**: read-write community が設定されておらず (`(none)`)、SNMP SET は不可

## 考察

1. **SwitchX-2 プラットフォームの制約**: SX6036 は SwitchX-2 ASIC を使用しており、MLNX-OS 3.6 が最終バージョン。新しい Spectrum ASIC (SN シリーズ) 向けの MLNX-OS でもファン速度制御は公開されておらず、これはプラットフォーム世代に関係なく Mellanox スイッチ共通の制約と考えられる

2. **温度連動制御**: ファン速度は温度センサーの値に基づいてファームウェアが自動調整している。ASIC 温度 32°C の状態で ~10000 RPM という値は、温度が低いにもかかわらず高めの RPM を維持していることを示す。冷却の安全マージンを大きく取る設計

3. **SNMP SET の非対応**: SNMP で fan sensor OID を取得できるが、書き込みは community 設定がないため不可。仮に read-write community を設定しても、ENTITY-SENSOR-MIB の sensor value は通常 read-only

4. **代替策**: ファン騒音を軽減したい場合、ファンモジュール (MSX60-FF) を低回転品に物理交換するか、PWM ファンコントローラーを外付けするしかない。ただし冷却能力の低下はスイッチの安定性に影響する可能性がある

## 再現方法

### シリアルコンソール接続

```sh
# スクリプト転送
scp -F ssh/config ./scripts/sx6036-console.py pve4:/tmp/

# ステータス取得
ssh -F ssh/config pve4 python3 /tmp/sx6036-console.py status

# show コマンド実行
ssh -F ssh/config pve4 python3 /tmp/sx6036-console.py show fan

# enable モードコマンド
ssh -F ssh/config pve4 python3 /tmp/sx6036-console.py enable-cmd show running-config

# configure モードでの試行 (oplog 必須)
echo "fsc mode manual" > /tmp/fan-test.txt
scp -F ssh/config /tmp/fan-test.txt pve4:/tmp/
./oplog.sh ssh -F ssh/config pve4 python3 /tmp/sx6036-console.py configure /tmp/fan-test.txt
```

### SNMP クエリ

```sh
# server 4 から SNMP で fan RPM 取得
ssh -F ssh/config pve4 snmpget -v2c -c public 10.10.10.100 \
    1.3.6.1.2.1.99.1.1.1.4.500020021 \
    1.3.6.1.2.1.99.1.1.1.4.500020022 \
    1.3.6.1.2.1.99.1.1.1.4.500020023 \
    1.3.6.1.2.1.99.1.1.1.4.500020024
```
