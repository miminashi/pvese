## 実施手順

### Phase 1: iso-download
- Debian 13.3 netinst ISO は既にダウンロード済み (sha256 検証 OK)

### Phase 2: preseed-generate
- iDRAC/R320 用 preseed (`preseed/preseed-server7.cfg`) は手動管理のため、generate-preseed.sh は不使用
- 確認のみ実施

### Phase 3: iso-remaster
- `remaster-debian-iso.sh` で preseed 埋め込み ISO を生成
- `--serial-unit=0` (R320 の SOL は COM1/ttyS0)
- EFI: Option B (grub-mkstandalone) で efi.img を再構築

### Phase 4: bmc-mount-boot
- iDRAC VirtualMedia で ISO をマウント (`//10.1.6.1/public/debian-preseed-s7.iso`)
- `boot-once VCD-DVD` 設定後、Power On

### Phase 5: install-monitor
- `sol-monitor.py` でパッシブ監視
- 約10分で完了 (LOADING_COMPONENTS -> CONFIGURING_APT -> INSTALLING_SOFTWARE -> INSTALLING_GRUB -> POWER_DOWN)
- インストーラが自動で poweroff

### Phase 6: post-install-config
- VirtualMedia unmount + boot-reset
- Power On -> SSH wait (210s timeout)
- 初回起動後、SSH はタイムアウト (PermitRootLogin 未設定のため)
- sol-login.py で root ログイン -> PermitRootLogin yes, SSH 鍵設置, 静的 IP 設定
- SSH 接続確認成功

### Phase 7: pve-install
- Step 0: pre-pve-setup.sh (DHCP + apt sources + route fix)
- Step 1: pve-setup-remote.sh --phase pre-reboot (PVE repo, kernel install, GRUB serial)
- Step 2: reboot -> SSH wait (170s)
- Step 3: pre-pve-setup.sh 再実行 (route fix)
- Step 4: pve-setup-remote.sh --phase post-reboot (proxmox-ve install, locale fix, Debian kernel removal)
- Step 5: reboot -> SSH wait (130s) -> route fix -> pveversion 確認

### Phase 8: cleanup
- VirtualMedia umount
- boot-reset
- 最終検証: OS, PVE, kernel, network, Web UI 全て OK

## トラブルシューティング

### GRUB インストール失敗 (UEFI モード)

**症状**: UEFI モードでブートした場合、grub-efi-amd64 パッケージがインストールされるが、
`grub-install /dev/sda` が "Unable to install GRUB in /dev/sda" で失敗する。

**原因**: R320 が UEFI BootMode に設定されていたため、Debian installer は UEFI GRUB
をインストールしようとしたが、partman の `atomic` レシピが ESP (EFI System Partition) を
正しく作成できなかった (あるいは `partman-efi/non_efi_system boolean true` が干渉した)。

**解決**: racadm で BootMode を `Bios` (Legacy) に切り替え:
```sh
ssh idrac7 racadm set BIOS.BiosBootSettings.BootMode Bios
ssh idrac7 racadm jobqueue create BIOS.Setup.1-1 -r pwrcycle -s TIME_NOW -e TIME_NA
```
切り替え後、Legacy BIOS 用 preseed (`grub-installer/bootdev string /dev/sda`) で再インストール。
grub-pc パッケージが正常にインストールされた。

**教訓**: R320 (iDRAC7) は UEFI モードでの GRUB インストールに問題がある。
Legacy BIOS モードを推奨する。preseed の `partman-efi/non_efi_system boolean true` は
Legacy BIOS モード専用の設定。
