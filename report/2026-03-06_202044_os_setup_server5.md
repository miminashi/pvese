## 前提・目的

5号機 (ayase-web-service-5) に Debian 13.3 (Trixie) と Proxmox VE 9.1.6 を BMC VirtualMedia 経由で自動インストールする。

- 背景: pvese プロジェクトの分散ストレージ評価環境を構築するために全ノードを統一した OS/PVE 環境にする
- 目的: 5号機を Debian 13.3 + PVE 9.1.6 で新規セットアップ
- 前提条件: BMC (Supermicro X11DPU) にネットワーク到達可能、SMB 共有にリマスター ISO を配置可能

## 再現方法

os-setup スキル (`./claude/skills/os-setup/SKILL.md`) に従い、Phase 1-8 を順次実行。

```sh
# Phase 1: ISO ダウンロード (既存ファイル再利用)
sha256sum /var/samba/public/debian-13.3.0-amd64-netinst.iso

# Phase 2: preseed 生成
./scripts/generate-preseed.sh config/server5.yml preseed/preseed-generated-s5.cfg

# Phase 3: ISO リマスター
./scripts/remaster-debian-iso.sh /var/samba/public/debian-13.3.0-amd64-netinst.iso \
    preseed/preseed-generated-s5.cfg /var/samba/public/debian-preseed-s5.iso

# Phase 4: BMC VirtualMedia マウント + ブート
./scripts/bmc-session.sh login 10.10.10.25 claude Claude123 tmp/82517841/bmc-cookie-s5
./scripts/bmc-virtualmedia.sh config/mount/verify ...
./scripts/bmc-power.sh cycle 10.10.10.25 claude Claude123 20
./scripts/bmc-power.sh find-boot-entry 10.10.10.25 claude Claude123 "ATEN Virtual CDROM"
# -> Boot0005
./scripts/bmc-power.sh boot-next 10.10.10.25 claude Claude123 Boot0005
./scripts/bmc-power.sh cycle 10.10.10.25 claude Claude123 20

# Phase 5: SOL インストール監視
./scripts/sol-monitor.py --bmc-ip 10.10.10.25 --bmc-user claude --bmc-pass Claude123 \
    --log-file tmp/82517841/sol-install-s5.log --max-reconnects 3

# Phase 6: SOL ログインで SSH 公開鍵 + 静的 IP 設定
./scripts/sol-login.py --bmc-ip 10.10.10.25 --bmc-user claude --bmc-pass Claude123 \
    --root-pass password --commands-file tmp/82517841/sol-commands-s5.txt

# Phase 7: PVE インストール
scp ./scripts/pve-setup-remote.sh root@10.10.10.205:/tmp/
ssh root@10.10.10.205 /tmp/pve-setup-remote.sh --phase pre-reboot --hostname ayase-web-service-5 --ip 10.10.10.205 --codename trixie --serial-unit 1
ssh root@10.10.10.205 reboot
# SSH 再接続待ち -> post-reboot
ssh root@10.10.10.205 /tmp/pve-setup-remote.sh --phase post-reboot --hostname ayase-web-service-5 --ip 10.10.10.205 --codename trixie --serial-unit 1
ssh root@10.10.10.205 reboot

# Phase 8: クリーンアップ + 最終検証
./scripts/bmc-virtualmedia.sh umount ...
./scripts/bmc-power.sh boot-override-reset ...
ssh root@10.10.10.205 pveversion
curl -sk https://10.10.10.205:8006
```

## 特記事項

- Phase 4 の POST code ポーリングで 0x00 到達後に 0x01 に戻る動作が観測された。サーバが POST 完了後にディスクからブートし、再度 POST サイクルに入った可能性がある。最終的に find-boot-entry でブートエントリ (Boot0005) を発見しセットアップ続行。
- Phase 6 での SSH 待機は静的 IP がまだ設定されていないためタイムアウトした (想定通り)。SOL ログインで直接静的 IP を設定後、即座に SSH 接続成功。
- POST code 0x92 のスタックは発生しなかった (5号機は安定)。
- インストーラのステージ進行: LOADING_COMPONENTS (3.7min) -> CONFIGURING_APT (3.9min) -> INSTALLING_SOFTWARE (7.1min) -> INSTALLING_GRUB (8.7min) -> PowerState Off (9.5min)
