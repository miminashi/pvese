## 前提・目的

- **背景**: PERC H710 の RAID 構成を VNC 経由で操作するスキルが必要。bios-setup スキル（Supermicro 4-6号機向け）と同様のパターンで、7-9号機の RAID セットアップを自動化する
- **目的**: VNC スクリーンショット + キーストローク方式で PERC BIOS Configuration Utility を操作するスキルを作成する
- **検証対象**: 8号機 (Dell PowerEdge R320, iDRAC7, PERC H710 Mini)

## 環境情報

- サーバ: 8号機 (10.10.10.28)
- iDRAC FW: 2.65.65.65
- PERC H710 Mini FW: 4.04-0001
- VNC: ポート 5901, パスワード Claude1, SSL 無効

## 調査・検証結果

### 1. VNC SYSTEM IDLE 問題の発見と解決

**問題**: iDRAC7 VNC はセッション切断後にビデオキャプチャが恒久的に停止する。

| 試行 | 結果 |
|------|------|
| VNCServer Disable/Enable | SYSTEM IDLE テキストは消えるが黒画面 |
| consolepreview API トリガー | 効果なし |
| 長時間キー送信 (F5 x10) | 効果なし |
| **racadm racreset** | **VNC ビデオキャプチャ完全復活** |

**発見**: racreset 後は独立した VNC 接続でも正常にスクリーンショットが取れる。

### 2. idrac-kvm-interact.py の作成

bmc-kvm-interact.py（Supermicro 用、Playwright ベース）に対応する VNC 版インタラクションスクリプトを作成。

| 項目 | bmc-kvm-interact.py | idrac-kvm-interact.py |
|------|---------------------|----------------------|
| 接続方式 | Playwright + HTML5 KVM | 直接 RFB ソケット |
| 依存 | Playwright, Chromium | cryptography, Pillow |
| レイテンシ | ~8秒 (ブラウザ起動) | ~1秒 (TCP 接続) |
| 対象 | 4-6号機 (Supermicro) | 7-9号機 (Dell iDRAC7) |

### 3. PERC BIOS メニュー構造の解明

| タブ | 内容 |
|------|------|
| VD Mgmt | VD/DG ツリー、VD 作成・削除 |
| PD Mgmt | PD 一覧、状態確認 |
| Ctrl Mgmt | コントローラ設定 |
| Properties | コントローラプロパティ |

**F2 Operations メニュー**:
- コントローラ行: Create New VD, Clear Config, Foreign Config 等
- VD 行: Initialization, Consistency Check, Delete VD, Properties 等

### 4. VD 操作の検証

**VD 削除**: 成功
- VD1 (data0, RAID-0, 837.75 GB) を F2 → Delete VD → YES で削除

**VD 作成フォームの操作**:
- RAID Level: Enter でドロップダウン → ArrowDown で選択 → Enter
- **PD 選択: ArrowDown で PD リストに移動 → Space でトグル** (Tab ではない)
- 選択状態: `[X]` + `#` 列に数字表示

### 5. VNC 接続の制約

| 制約 | 影響 | 対策 |
|------|------|------|
| SYSTEM IDLE | VNC 切断後に復帰しない | racreset (90-120秒) |
| 解像度変更 | POST→PERC で接続ドロップ | Ctrl+R は screenshot なし |
| タイムアウト | 長時間操作で切断 | 短いバッチに分割 |

## 再現方法

### PERC BIOS への進入

```sh
# 1. racreset で VNC 復活
./oplog.sh ssh -F ssh/config idrac8 racadm racreset
sleep 120
ssh-keygen -R 10.10.10.28 -f ssh/known_hosts

# 2. Power cycle
./oplog.sh ipmitool -I lanplus -H 10.10.10.28 -U claude -P Claude123 chassis power cycle
sleep 20

# 3. Ctrl+R (screenshot なし)
python3 ./scripts/idrac-kvm-interact.py --bmc-ip 10.10.10.28 --timeout 60 sendkeys Ctrl+r x20 --wait 2000

# 4. スクリーンショット確認
python3 ./scripts/idrac-kvm-interact.py --bmc-ip 10.10.10.28 screenshot tmp/perc.png
```
