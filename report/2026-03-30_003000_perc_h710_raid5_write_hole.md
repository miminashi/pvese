# PERC H710 Mini RAID5 ライトホール問題調査

- 作業日時: 2026-03-30 JST
- 対象: 7号機・8号機・9号機 (Dell PowerEdge R320, PERC H710 Mini FW 21.3.0-0009)

## 目的

7-9号機で RAID5 を構成した場合に、BBU (Battery Backup Unit) 消耗時にライトホール問題が生じるか、およびライトホールを回避する設定が可能かを調査する。

## 背景: RAID5 ライトホールとは

RAID5 のストライプ書き込みは、データブロックとパリティブロックを複数のディスクに非アトミックに書き込む。書き込み途中で電源断が発生すると、データとパリティの整合性が崩れる（ライトホール）。この状態でディスク故障が重なると、パリティからの復元が不正確になりデータロスにつながる。

## 現在の BBU 状態

| サーバ | BBU Status | BBU State | LearnMode | キャッシュ |
|--------|-----------|-----------|-----------|-----------|
| 7号機 | Ok | Ready | Not supported | 512 MB |
| 8号機 | Ok | Ready | Not supported | 512 MB |
| 9号機 | Ok | Ready | Not supported | 512 MB |

- BBU タイプ: リチウムイオン TBBU (Transportable Battery Backup Unit)
- Dell パーツ番号: 70K80 互換 (3.7V / 500mAh / 1.8Wh)
- `LearnMode = Not supported`: PERC H710 Mini (Embedded) では Learn Cycle の手動トリガーが iDRAC 経由でサポートされていない。90日間隔の自動 TLC (Transparent Learn Cycle) は FW 側で実行される

## 現在の Write Policy

全 VD が **Write Back** で動作中 (dmesg: `Write cache: enabled`)。

| サーバ | VD0 (RAID-1) | VD1-VDn (RAID-0) |
|--------|-------------|-----------------|
| 7号機 | Write Back | Write Back |
| 8号機 | Write Back | Write Back |
| 9号機 | Write Back | Write Back |

## 調査結果

### Q1: BBU 消耗時にライトホールは発生するか

**はい、発生する。**

PERC H710 の RAID5 ライトホール対策は **BBU によるキャッシュ保護のみ**に依存している。

| BBU 状態 | Write Policy | ライトホールリスク | 説明 |
|----------|-------------|-------------------|------|
| 正常 | Write Back | **低** | ストライプ全体がキャッシュに蓄積され、BBU が電源断時にキャッシュを保護。復電後にフラッシュ |
| 消耗/故障 | **自動で Write Through に切替** | **高** | キャッシュなしで書き込み。データとパリティの非アトミック書き込みが直接ディスクへ → 電源断でライトホール発生 |
| 消耗 + Force WB | Write Back (強制) | **極めて高** | キャッシュ使用するが電源断でキャッシュ内容消失 → ライトホール + キャッシュロスの二重リスク |

**PERC H710 には以下のライトホール対策機能がない:**
- ジャーナルベースのパリティログ (Partial Parity Log / PPL)
- Write-intent bitmap
- 非揮発性キャッシュへのパリティジャーナリング
- ハードウェア RAID5 write hole protection (新世代コントローラの機能)

### Q2: BBU 消耗時のライトホール回避設定は可能か

**ハードウェア RAID レベルでは不可能。** ソフトウェアレベルでの緩和策のみ。

#### PERC H710 で利用可能な設定

| 設定 | racadm コマンド | 効果 | ライトホール対策 |
|------|---------------|------|----------------|
| Write Through | VD 作成時 `-wp wt` | キャッシュ不使用。性能低下 | **対策にならない** (WT でもデータ+パリティの非アトミック書き込みは発生) |
| Write Back | VD 作成時 `-wp wb` (デフォルト) | BBU 正常時のみ安全 | BBU 正常時のみ有効 |
| Force Write Back | PERC BIOS で設定 | BBU 不在でも WB 強制 | **危険**。UPS 必須 |
| Patrol Read | `racadm raid patrolread` | 定期的にディスク全面読取 | ライトホール予防にはならないが、サイレント故障の早期検出に有効 |
| Check Consistency | `racadm raid ccheck` | パリティ整合性検査 | ライトホール発生後の検出・修復に有効（予防ではない） |

#### Write Through でもライトホールは防げない理由

Write Through はキャッシュをバイパスするが、RAID5 のストライプ書き込みでは物理的に複数ディスクへの書き込みが非アトミックに行われる。Write Through は「キャッシュロス」は防ぐが「部分ストライプ書き込み」は防げない。

### Q3: ソフトウェアレベルの緩和策

| 対策 | 実現方法 | 効果 |
|------|---------|------|
| **UPS 導入** | 外部 UPS でサーバ全体を保護 | 電源断自体を防止 → ライトホール発生機会を排除 |
| **md RAID5 + bitmap** | Linux md で `--bitmap=internal` 付き RAID5 | Write-intent bitmap で部分書き込みを追跡。復旧時にビットマップ範囲のみ再同期 |
| **md RAID5 + PPL** | Linux md で `--consistency-policy=ppl` | Partial Parity Log でストライプ書き込みをジャーナリング。ライトホールをほぼ完全に防止 |
| **DRBD による保護** | 現在の LINSTOR/DRBD 構成を維持 | DRBD の application-level replication がディスク障害 + ライトホールの組み合わせからの復旧手段を提供 |
| **RAID-1 または RAID-10 使用** | PERC RAID-1/10 | ライトホール問題が構造的に発生しない (パリティ計算なし) |

## 結論と推奨

### PERC H710 で RAID5 を組む場合のリスク

1. **BBU 正常時**: Write Back キャッシュが電源断からストライプ書き込みを保護。ライトホールリスクは低い
2. **BBU 消耗時**: 自動的に Write Through に切り替わるが、WT でもライトホールは防げない。PERC H710 にはジャーナルベースの保護機構がないため、**BBU 消耗 = ライトホールに対して無防備**
3. **BBU の寿命**: リチウムイオン TBBU の一般的な寿命は 2-4 年。7-9号機は中古サーバのため、BBU の残寿命は不明

### 推奨

| 推奨 | 理由 |
|------|------|
| **PERC RAID5 は使用しない** | PERC H710 にはライトホール保護機構がなく、BBU 消耗時に脆弱。現在の RAID-0 個別 VD + LINSTOR/DRBD レプリケーションの方が安全 |
| **RAID5 が必要なら Linux md + PPL を使用** | `--consistency-policy=ppl` でソフトウェア側でライトホールを防止。PERC は JBOD または RAID-0 パススルーで使用 |
| **BBU を定期監視する** | `racadm storage get batteries -o` で Status/State を確認。Ok/Ready 以外なら即交換 |
| **UPS の導入を検討** | BBU 状態に依存しない根本的な電源断対策 |
