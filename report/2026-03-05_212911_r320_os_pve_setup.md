## R320 固有の知見

### SOL が使えない
R320 の SOL は Linux シリアル出力を通さないため、preseed には `console=tty0` を設定。Phase 6 の SSH 鍵設定は pexpect で debian ユーザにパスワードログインし、su で root に切り替えて実施する。

### preseed は CD-only (apt mirror なし)
Minimal install のため wget, ca-certificates が未インストール。PVE インストール前に以下が必要:
1. apt sources.list に Debian ミラーを追加
2. `apt-get install wget ca-certificates`

### インターネットアクセス
- 10.0.0.0/8 はインターネット不可（内部通信専用）
- 192.168.39.0/24 (DHCP) 経由でインターネット接続
- `/etc/network/interfaces` に gateway 10.10.10.1 があるため、リブートのたびに `ip route del default via 10.10.10.1` が必要
- DHCP IPv4 割当に 5-10秒かかることがある

### POST 時間
- Lifecycle Controller: Collecting System Inventory で 2-3分
- リブート後の SSH 到達まで約 3.5分

## 再現方法

### 前提
- iDRAC SSH 鍵認証設定済み (`~/.ssh/idrac_rsa`)
- SMB 共有に Debian 13.3 ISO が配置済み (`/var/samba/public/`)

### コマンド

```bash
# Phase 1: ISO SHA256 検証
sha256sum /var/samba/public/debian-13.3.0-amd64-netinst.iso

# Phase 3: ISO リマスター
./scripts/remaster-debian-iso.sh --legacy-only /var/samba/public/debian-13.3.0-amd64-netinst.iso preseed/preseed-server7.cfg /var/samba/public/debian-preseed.iso

# Phase 4: VirtualMedia マウント + ブート
./scripts/idrac-virtualmedia.sh mount 10.10.10.120 //10.1.6.1/public/debian-preseed.iso guest guest
./scripts/idrac-virtualmedia.sh boot-once 10.10.10.120 VCD-DVD
ipmitool -I lanplus -H 10.10.10.120 -U claude -P Claude123 chassis power on

# Phase 5: VNC で監視 (約13分で SYSTEM POWER OFF)
.venv/bin/python3 tmp/<session>/vnc-wake-screenshot.py --host 10.10.10.120 --port 5901 --password Claude1 --output screenshot.png

# Phase 6: umount + boot-reset + 電源 ON + SSH 設定
./scripts/idrac-virtualmedia.sh umount 10.10.10.120
./scripts/idrac-virtualmedia.sh boot-reset 10.10.10.120
ipmitool -I lanplus -H 10.10.10.120 -U claude -P Claude123 chassis power on
# (3.5分待機)
# pexpect で debian ユーザ経由 SSH 鍵設定 (tmp/<session>/ssh-setup.py)

# Phase 7: PVE インストール
scp tmp/<session>/pre-pve-setup.sh root@10.10.10.207:/tmp/
ssh root@10.10.10.207 sh /tmp/pre-pve-setup.sh
scp ./scripts/pve-setup-remote.sh root@10.10.10.207:/tmp/
ssh root@10.10.10.207 /tmp/pve-setup-remote.sh --phase pre-reboot --hostname ayase-web-service-7 --ip 10.10.10.207 --codename trixie
ssh root@10.10.10.207 reboot
# (3.5分待機)
ssh root@10.10.10.207 ip route del default via 10.10.10.1
scp ./scripts/pve-setup-remote.sh root@10.10.10.207:/tmp/
ssh root@10.10.10.207 /tmp/pve-setup-remote.sh --phase post-reboot --hostname ayase-web-service-7 --ip 10.10.10.207 --codename trixie
ssh root@10.10.10.207 reboot
# (3.5分待機)

# Phase 8: 最終検証
ssh root@10.10.10.207 pveversion
ssh root@10.10.10.207 uname -r
curl -sk -o /dev/null -w '%{http_code}' https://10.10.10.207:8006
```
