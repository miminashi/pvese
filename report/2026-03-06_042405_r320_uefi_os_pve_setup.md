# R320 (7号機) UEFI モード OS 再インストール + PVE セットアップ

- **実施日時**: 2026年3月6日 04:24
- **関連レポート**: [report/2026-03-05_212911_r320_os_pve_setup.md](2026-03-05_212911_r320_os_pve_setup.md) (前回 Legacy BIOS 試行)
- **関連 Issue**: #31

## 前提・目的

SOL (Serial Over LAN) 対応のための以下の変更を通しテストで検証する:

- `preseed/preseed-server7.cfg`: `console=ttyS0,115200n8` 追加
- `scripts/remaster-debian-iso.sh`: `--serial-unit=N` パラメータ追加
- `scripts/pve-setup-remote.sh`: `grep -q 'console=ttyS'` に修正 (ttyS0 上書き防止)

### 背景: Legacy BIOS → UEFI 切り替え

前回の通しテスト (2026-03-05) で preseed インストール自体は成功するが、HDD ブートで "No boot device available" が発生した。原因は **Debian 13 が GPT パーティションテーブルをデフォルトで作成し、R320 の Legacy BIOS が GPT からブートできない**こと。ユーザの提案により UEFI モードに切り替えて解決した。

## 発見・修正事項

### 1. UEFI モード切替手順

```sh
# Legacy BIOS → UEFI
ssh -i ~/.ssh/idrac_rsa claude@10.10.10.120 racadm set BIOS.BiosBootSettings.BootMode Uefi
ssh -i ~/.ssh/idrac_rsa claude@10.10.10.120 racadm jobqueue create BIOS.Setup.1-1 -r pwrcycle -s TIME_NOW -e TIME_NA
# Power On して JOB 完了を待つ (約6分)
```

### 2. remaster-debian-iso.sh Option B embed.cfg 修正 (バグ修正)

EFI standalone GRUB の embed.cfg に `preseed/file=/cdrom/preseed.cfg` が欠落していた。
UEFI ブートでインストーラが preseed を読まず、対話ダイアログが表示される問題を修正。

修正前:
```
linux /install.amd/vmlinuz auto=true priority=critical locale=en_US.UTF-8 keymap=us console=tty0 console=ttyS${SERIAL_UNIT},115200n8 ---
```

修正後:
```
linux /install.amd/vmlinuz vga=normal nomodeset auto=true priority=critical preseed/file=/cdrom/preseed.cfg locale=en_US.UTF-8 keymap=us console=tty0 console=ttyS${SERIAL_UNIT},115200n8 --- quiet
```

### 3. racadm set vs racadm config (iDRAC7)

`racadm set iDRAC.ServerBoot.BootOnce Disabled` は iDRAC7 FW 2.65.65.65 で**サイレントに失敗**する。"Object value modified successfully" を返すが値は変更されない。

修正: `racadm config -g cfgServerInfo -o cfgServerBootOnce 0` (legacy コマンド) を使用。

注意: `FirstBootDevice` は `racadm set` が機能する。`BootOnce` のみ legacy コマンドが必要。

### 4. Debian 13 + GPT + Legacy BIOS の非互換性

Debian 13 (Trixie) インストーラは 2TB 未満のディスクでもデフォルトで GPT パーティションテーブルを作成する。R320 の Legacy BIOS は GPT からブートできないため "No boot device available" が発生する。

解決策: UEFI モードに切り替える (上記手順)。

### 5. pre-pve-setup で DHCP IPv4 取得に時間がかかる

`ifup eno2` 後、IPv6 は即座に取得されるが IPv4 DHCP は数秒〜十数秒遅延する。`sleep 5` では不足する場合がある。スクリプトでは apt-get 失敗後にリトライで対処した。

## 再現方法

### Phase 3: ISO リマスター

```sh
./scripts/remaster-debian-iso.sh /var/samba/public/debian-13.3.0-amd64-netinst.iso \
  preseed/preseed-server7.cfg \
  /var/samba/public/debian-preseed.iso \
  --serial-unit=0
```

`--legacy-only` を付けない (UEFI + Legacy dual boot ISO)。

### Phase 4: VirtualMedia マウント + BootOnce

```sh
ssh -i ~/.ssh/idrac_rsa claude@10.10.10.120 racadm remoteimage -c -l //10.1.6.1/public/debian-preseed.iso -u guest -p guest
ssh -i ~/.ssh/idrac_rsa claude@10.10.10.120 racadm config -g cfgServerInfo -o cfgServerBootOnce 1
ssh -i ~/.ssh/idrac_rsa claude@10.10.10.120 racadm set iDRAC.ServerBoot.FirstBootDevice VCD-DVD
ssh -i ~/.ssh/idrac_rsa claude@10.10.10.120 racadm serveraction powercycle
```

### Phase 7: PVE インストール (pre-pve-setup)

```sh
# pre-pve-setup.sh: DHCP on eno2, default route fix, apt sources, wget/ca-certificates
scp tmp/<session-id>/pre-pve-setup.sh root@10.10.10.207:/tmp/
ssh root@10.10.10.207 sh /tmp/pre-pve-setup.sh
# IPv4 DHCP 取得に失敗した場合、手動で apt-get update + install を再実行
```
