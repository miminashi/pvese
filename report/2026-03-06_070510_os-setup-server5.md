## 問題と対応

### Phase 5 初回失敗: preseed が完全適用されない

**症状**: Debian インストーラが medium priority に落ちて main menu を表示し、ユーザセットアップで対話式入力を要求した。

**原因**: カーネルコマンドラインに `netcfg/choose_interface=auto` が含まれていなかったため、10GbE NIC (eno1np0) にリンクがない状態で d-i がネットワークインターフェース選択のプロンプトを出し、priority が medium に降格した。一度 medium になると、後続の preseed 値が自動適用されなくなる。

**修正**: `scripts/remaster-debian-iso.sh` のカーネルコマンドライン (GRUB, txt.cfg, EFI embed の3箇所) に `netcfg/choose_interface=auto` を追加。

**結果**: 修正後の再インストールでは、全ステップが完全自動で進行し、約9分で完了した。

## 再現方法

```sh
# Phase 1-2: ISO download + preseed generate
./scripts/os-setup-phase.sh init --config config/server5.yml
./scripts/generate-preseed.sh config/server5.yml preseed/preseed-generated-s5.cfg

# Phase 3: ISO remaster (netcfg/choose_interface=auto が kernel cmdline に含まれること)
./scripts/remaster-debian-iso.sh /var/samba/public/debian-13.3.0-amd64-netinst.iso \
    preseed/preseed-generated-s5.cfg /var/samba/public/debian-preseed-s5.iso

# Phase 4: BMC mount + boot
# (bmc-session.sh login, bmc-virtualmedia.sh config/mount, bmc-power.sh boot-next/cycle)

# Phase 5: SOL passive monitor
./scripts/sol-monitor.py --bmc-ip 10.10.10.25 --bmc-user claude --bmc-pass Claude123 \
    --log-file tmp/38ce6813/sol-install-s5.log

# Phase 6: SOL login + post-install config
./scripts/sol-login.py --bmc-ip 10.10.10.25 --bmc-user claude --bmc-pass Claude123 \
    --root-pass password --commands-file tmp/38ce6813/sol-commands-s5.txt

# Phase 7: PVE install
ssh root@10.10.10.205 /tmp/pve-setup-remote.sh --phase pre-reboot --hostname ayase-web-service-5 --ip 10.10.10.205 --codename trixie
ssh root@10.10.10.205 reboot
# (wait for PVE kernel boot)
ssh root@10.10.10.205 /tmp/pve-setup-remote.sh --phase post-reboot --hostname ayase-web-service-5 --ip 10.10.10.205 --codename trixie
ssh root@10.10.10.205 reboot

# Phase 8: cleanup (VirtualMedia umount, boot-override-reset, verify)
```

## 変更したファイル

- `scripts/remaster-debian-iso.sh` - カーネルコマンドラインに `netcfg/choose_interface=auto` を追加 (GRUB, txt.cfg, EFI embed の3箇所)
- `config/server5.yml` - `iso_filename` を一時的に `debian-preseed-s5.iso` に変更し、完了後 `debian-preseed.iso` に復元
